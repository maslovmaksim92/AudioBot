<product_requirements>
The VasDom AudioBot is a cleaning operations management application. The core requirement is to enable real-time, AI-powered voice interactions. The initial implementation targeted outbound calls initiated by an AI agent, leveraging LiveKit for WebRTC and Novofon for SIP termination.

A critical constraint emerged: Novofon's SIP service requires the calling infrastructure to be located in Russia. This forced an architectural pivot. The current implementation addresses this by using a dedicated Virtual Machine on Yandex Cloud (a Russian provider) to host an Asterisk server. This server acts as a SIP bridge, registering with Novofon from a permitted IP address.

The complete call flow is now: Frontend UI -> Backend API -> LiveKit Cloud -> LiveKit SIP Gateway -> Asterisk SIP Bridge (on Yandex VM) -> Novofon SIP Trunk -> Public Switched Telephone Network (PSTN).

The immediate next feature is to integrate a specific OpenAI Realtime prompt () into the call flow, enabling the AI to conduct conversations based on this prompt when a call is initiated from the application's frontend.
</product_requirements>
