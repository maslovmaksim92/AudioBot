<product_requirements>
The primary goal is to build a feature enabling the VasDom AudioBot to make outbound calls to PSTN numbers, where an AI voice agent handles the conversation in real-time. The AI should use a specific OpenAI prompt ().

A core architectural constraint imposed by the SIP provider (Novofon) is that all SIP traffic must originate from a Russian IP address. This led to a multi-stage setup:
1. A React frontend () initiates a call via the backend.
2. The FastAPI backend orchestrates the creation of a LiveKit room and a SIP participant.
3. LiveKit's SIP Gateway routes the call to a custom Asterisk server on a Yandex Cloud VM in Russia.
4. The Asterisk server bridges the call to the PSTN via the Novofon SIP trunk.
5. The backend runs a worker that connects directly to OpenAI's Realtime API via WebSockets, bridging the audio between the PSTN participant and the AI within the LiveKit room.

The feature is partially functional. Calls connect successfully, and audio from the PSTN is being forwarded to OpenAI. However, the AI does not respond to the user's voice due to an audio buffering issue.
</product_requirements>
