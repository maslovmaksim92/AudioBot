The goal is to integrate an AI-powered, real-time voice agent into the VasDom AudioBot application for making outbound calls. The AI should use a specific OpenAI prompt () and initiate the conversation upon connection.

A key technical constraint is that the SIP provider, Novofon, requires all SIP traffic to originate from a Russian IP address. This led to the current architecture:
1.  A React frontend () initiates the call.
2.  A FastAPI backend () orchestrates the process.
3.  The backend uses the LiveKit Cloud API to create a SIP participant.
4.  LiveKit's SIP Gateway routes the call to a custom Asterisk server hosted on a Yandex Cloud VM (in Russia).
5.  The Asterisk server acts as a bridge, forwarding the call via its registered Novofon SIP trunk to the Public Switched Telephone Network (PSTN).
6.  Simultaneously, the backend connects an AI agent directly to OpenAI's Realtime API via WebSockets and bridges the audio streams within the LiveKit room.

The feature is partially implemented: the call connects, but there is silence instead of an AI conversation.
