<product_requirements>
The VasDom AudioBot centralizes management for a cleaning company, integrating Bitrix24 CRM, an AI assistant (Emergent LLM), a Telegram bot, and a dashboard for reporting. The core problem is streamlining the management of cleaning objects (houses), tasks, employees, and communication.

Key feature enhancements include:
1.  **Works (Houses) Tab**: Display 490 houses from Bitrix24's Уборка подъездов (ID 34) funnel. House cards show address, apartments, entrances, floors, correct brigade name, management company, and September's cleaning schedule. Pagination (50 houses/page) is required. Buttons for a full Schedule modal (September-December 2025) and Details modal (full house, company, and contact info) are implemented, along with a Периодичность (Periodicity) field based on cleaning frequency.
2.  **Dashboard**: Multiple enhancements, including quick filters (brigade, management company, date range), Today block with real-time Bitrix data, status graphs, and compact KPI cards.
3.  **AI Chat**: Talk Online button for real-time voice conversation using OpenAI Realtime (STT+TTS) with natural, non-robotic voice. Responses should be both text and voice.
4.  **Meetings (Планёрка)**: Online transcription (live) using OpenAI Realtime, summarization, and export functionality.
5.  **AI Training (Обучение АИ)**: Ability to upload files, AI processes content, provides a summary (what AI understood), user confirms/supplements, and it's added to a knowledge base using PostgreSQL + pgvector.
6.  **AI Tasks (Задачи для АИ)**: Users can set tasks (e.g., send report tomorrow 12:00 in Telegram), which are then managed by a background scheduler () and trigger Telegram notifications.
7.  **Logistics**: Implementation of features for calculating optimal routes and logistics based on cleaning schedules and addresses, leveraging Google Maps Directions API.
8.  **UI/UX Refinements**: Elimination of empty space (padding/margins) across the dashboard, consistent component spacing, and overall design improvements to match a desired aesthetic.
</product_requirements>
