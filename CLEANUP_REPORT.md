## Детальный анализ

### 1. Текущее состояние `/app`

#### Backend (`/app/backend/`)
```
backend/
├── .env                # Переменные окружения (MONGO_URL, DB_NAME)
├── requirements.txt    # Минимальные зависимости (FastAPI, Motor)
└── server.py          # Базовый FastAPI сервер (70 строк)
```

**Найденный код:**
- Простой FastAPI app с 2 эндпоинтами (`/api/`, `/api/status`)
- MongoDB подключение через Motor
- Pydantic модель `StatusCheck`
- CORS middleware

**Отсутствует:**
- Модульная структура (`app/routers`, `app/services`, `app/config`)
- Интеграции (Bitrix24, OpenAI, LiveKit, Telegram)
- База знаний (RAG с pgvector)
- Планировщик задач (APScheduler)
- Полная функциональность VasDom

#### Frontend (`/app/frontend/`)
```
frontend/
├── .env                       # REACT_APP_BACKEND_URL
├── package.json               # React, TailwindCSS, shadcn/ui
├── src/
│   ├── App.js                 # Минимальный компонент (54 строки)
│   ├── index.js
│   ├── App.css, index.css
│   ├── components/ui/         # shadcn/ui компоненты (50+ файлов)
│   ├── hooks/use-toast.js
│   └── lib/utils.js
└── public/index.html
```

**Найденный код:**
- Базовый React app с маршрутизацией
- Один route `/` с приветственным экраном
- Пример API вызова к `/api/`
- shadcn/ui компоненты (не используются)

**Отсутствует:**
- Компоненты приложения (Dashboard, Works, AI Chat, Meetings, Training, и т.д.)
- Роутинг к страницам приложения
- Интеграция с backend API VasDom
- Вся бизнес-логика VasDom

### 1. Нет проблемы дубликатов
**Вывод:** Текущий рабочий каталог `/app` чист и не содержит дублирующих файлов. Проблема дубликатов отсутствует.

### 2. Несоответствие ожиданиям
**Проблема:** Текущий код - это **стартовый шаблон**, а не **полное приложение VasDom** из репозитория GitHub.

**Возможные причины:**
- Предыдущие AI агенты создали базовый шаблон вместо интеграции полного кода
- Код в GitHub не был перенесён в рабочий каталог
- Разработка ещё не завершена

### 3. Что нужно сделать?

**Опция А: Использовать текущий минимальный код**
- Текущий код работает как базовый API starter
- Подходит для начала нового проекта
- Требует полной разработки всех функций VasDom

**Опция Б: Интегрировать код из GitHub**
- Получить полную функциональность VasDom
- Требует тщательного переноса и настройки
- Нужно настроить переменные окружения
- Нужно PostgreSQL вместо MongoDB
