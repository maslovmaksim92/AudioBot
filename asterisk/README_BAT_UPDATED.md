## 🔧 Порядок действий для тестирования

### Шаг 1: Заблокируйте атаки (один раз)
```
Запустите: block_attackers.bat
```

### Шаг 2: Запустите мониторинг
```
Запустите: monitor_clean.bat
```

### Шаг 3: Сделайте звонок
- Откройте приложение
- Нажмите "AI Звонок"
- Введите: +79200924550
- Нажмите кнопку звонка

### Шаг 4: Смотрите результат
В окне PowerShell вы должны увидеть:
```
[2025-10-02 xx:xx:xx] VERBOSE[xxxxx] res_pjsip_session.c: livekit-endpoint: Method is INVITE
[2025-10-02 xx:xx:xx] DEBUG[xxxxx] res_pjsip_session.c: livekit-endpoint: Response is 100 Trying
```

## 🆘 Если ничего не появляется

### 1. Проверьте LiveKit Dashboard
- Address должен быть: `51.250.74.43:5060`
- Transport: UDP
- Username/Password: пусто (без аутентификации)

### 2. Попробуйте звонок на 8888 (эхо)
Это тестовый номер, должен работать всегда.

### 3. Проверьте firewall
Запустите: `quick_connect.bat`
Затем выполните:
```bash
sudo ufw status
```

Должны быть разрешены порты:
- 5060/udp (SIP)
- 10000:20000/udp (RTP)

## 💡 Полезные команды на сервере

После подключения через `quick_connect.bat`:

```bash
# Стать root
sudo su

# Проверить endpoints
asterisk -rx "pjsip show endpoints"

# Перезагрузить конфигурацию
asterisk -rx "pjsip reload"

# Посмотреть активные звонки
asterisk -rx "core show channels"

# Посмотреть регистрации
asterisk -rx "pjsip show registrations"
```
